# Member Roles — Catalog

## Overview

Per-organization role definitions and the role ↔ permission join table. A `member_role` is a named bundle of permissions an organization owner can assign to memberships (e.g. "Owner", "Admin", "Viewer"). The nested [member-role-permission](src/domains/tenancy/sub-domains/member-roles/member-role-permission/) resource is the join table that ties roles to the catalog of permission constants.

| POST | `/api/v1/tenancy/organizations/:id/roles` | PERM: role:manage | Creates a new custom role in the organization. Requires ROLE_MANAGE permission. |
| DELETE | `/api/v1/tenancy/organizations/:id/roles/:roleId` | PERM: role:manage | Deletes a custom role. System roles cannot be deleted. Members with this role must be reassigned first. Requires ROLE_MANAGE permission. |
| GET | `/api/v1/tenancy/organizations/:id/roles/:roleId` | PERM: role:read | Returns a single role with its details. Requires ROLE_READ permission. |
| PATCH | `/api/v1/tenancy/organizations/:id/roles/:roleId` | PERM: role:manage | Updates a role name or description. System roles cannot be modified. Requires ROLE_MANAGE permission. |
| GET | `/api/v1/tenancy/organizations/:id/roles/:roleId/permissions` | PERM: role:read | Returns all permissions assigned to a role. Requires ROLE_READ permission. |
| PUT | `/api/v1/tenancy/organizations/:id/roles/:roleId/permissions` | PERM: role:manage | Replaces all permissions for a role with the provided set. Requires ROLE_MANAGE permission. |
