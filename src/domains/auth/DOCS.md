# Auth — Catalog

## Overview

Owns identity proof and session establishment for the platform. Supports five primary credential types — passwords (argon2id), magic links, OAuth, MFA (TOTP), and WebAuthn — and issues short-lived JWTs (RS256) backed by refreshable session rows. The domain is the source of truth for "who is making this request"; everything authorization-related (organization permissions) lives in [tenancy](src/domains/tenancy/).

### `MagicLinkSendResult`

Source: `src/domains/auth/auth.types.ts`

Magic-link send result.

The raw token is intentionally absent — it is dispatched only via
`AUTH_EVENT.MAGIC_LINK_REQUESTED` and the resulting email. Tests use
`captureNextMagicLinkToken` (src/tests/helpers/magic-link.helper.ts) to read it.

`messageKey` is translated by the HTTP handler via `translateMessageKeyPayload`
before reaching the API response.
