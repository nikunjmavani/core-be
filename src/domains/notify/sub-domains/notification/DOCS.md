# Notification — Catalog

## Overview

In-app notification rows: the user-visible feed that surfaces "subscription went past due", "you were invited to organization X", etc. Cross-domain event handlers create rows here; users mark them read or dismiss them via the API.

→ Read the full overview: [OVERVIEW.md](./OVERVIEW.md)

## Routes

| Method | Path | Access | Description |
| --- | --- | --- | --- |
| GET | `/api/v1/notify/notifications` | AUTH | Returns a paginated list of notifications for the authenticated user. |
| GET | `/api/v1/notify/notifications/:id` | AUTH | Returns a single notification by ID. |
| PATCH | `/api/v1/notify/notifications/:id/read` | AUTH | Marks a single notification as read. |
| DELETE | `/api/v1/notify/notifications/:notificationId` | AUTH | Permanently deletes a notification. |
| POST | `/api/v1/notify/notifications/mark-all-read` | AUTH | Marks all unread notifications as read for the authenticated user. |
| GET | `/api/v1/notify/notifications/unread-count` | AUTH | Returns the count of unread notifications for the authenticated user. |
